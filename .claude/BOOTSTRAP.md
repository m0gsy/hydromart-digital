# BOOTSTRAP

You are joining an existing production software project.

Your first responsibility is to understand the system before making any changes.

Never begin implementation before completing project discovery.

======================================================
PROJECT DISCOVERY
======================================================

Read the entire repository.

Including but not limited to

- README
- BRD
- PRD
- Architecture Documentation
- API Documentation
- Environment Configuration
- Package Configuration
- Database Schema
- Docker Configuration
- CI/CD Pipelines
- Existing Tests
- Coding Standards
- Claude Memory

Analyze

- Folder Structure
- Module Structure
- Dependencies
- Shared Libraries
- Existing Features
- Coding Patterns
- Naming Conventions

======================================================
GENERATE
======================================================

Generate

- Architecture Graph
- Dependency Graph
- Module Graph
- Database Relationship Graph
- Feature Dependency Graph

======================================================
IDENTIFY
======================================================

Identify

- Business Rules
- Acceptance Criteria
- Bounded Contexts
- Domains
- Existing Design Patterns
- Technical Debt
- Security Risks
- Performance Bottlenecks
- Existing Coding Standards
- Current Milestone

======================================================
PROJECT MEMORY
======================================================

Store into Claude Memory

- Architecture Decisions
- Folder Structure
- Module Relationships
- Business Rules
- API Contracts
- Naming Conventions
- Technical Debt
- Current Progress
- Open Issues

======================================================
IMPLEMENTATION RULES
======================================================

Never write code before understanding the project.

Never rewrite working code without strong engineering justification.

Prefer extending existing implementations over replacing them.

Maintain architectural consistency.

Reuse existing patterns whenever possible.

Document every assumption before implementation.

======================================================
BOOTSTRAP COMPLETE
======================================================

Bootstrap is complete only when

✓ Repository understood

✓ Business domain understood

✓ Architecture understood

✓ Current milestone identified

✓ Claude Memory updated

Only then continue implementation.
