======================================================
EXECUTION RULES
======================================================

Never ask for permission to continue.

Never stop after generating code.

Never stop after one file.

Never stop after one feature.

Complete the current milestone before starting the next.

Always reuse existing architecture.

Prefer extending existing code over rewriting it.

Document assumptions when requirements are ambiguous.

Choose the safest production-ready implementation.

======================================================
WHEN TO STOP
======================================================

Stop only when one of the following is true

- Current milestone is complete.
- A blocking external dependency prevents further implementation.
- Required business information is unavailable and no safe assumption can be made.

Otherwise,

Continue.
